### Overview of Science ASimulation

Science ASimulation is an innovative, web-based educational platform designed to revolutionize STEM learning, particularly in biology and chemistry, through immersive virtual simulations. Built on a modern Next.js stack (with TypeScript, React Three Fiber for 3D rendering, and integrations like Mol* for molecular visualization), it serves as a safe, accessible "virtual laboratory" for students, educators, and researchers. Launched in late 2025, it draws inspiration from established tools like PhET Interactive Simulations but extends them with AI-driven personalization, user workspaces, and real-time reaction modeling using libraries like RDKit-js for chemistry and BioPython-inspired biology pathways.

The platform's core mission is to bridge theoretical knowledge with hands-on practice, eliminating barriers like equipment costs, safety risks, or lab access. It's free for basic use (with premium tiers for advanced features via Supabase backend), browser-compatible (no downloads needed), and optimized for schools, universities, and self-learners. As of November 25, 2025, it hosts over 150 simulations, with expansions planned for physics and environmental science. User data is GDPR-compliant, with optional progress syncing via Google or email login.

Key stats (from platform analytics):
- **Users**: 50,000+ active (mostly K-12 and undergrad)
- **Simulations Run**: 1M+ sessions
- **Accessibility**: WCAG 2.1 AA compliant, multilingual (English, Spanish, French; more in beta)

### All Functionalities

Science ASimulation emphasizes interactivity and data-driven learning. Core functionalities include:

1. **Simulation Engine**:
   - Real-time 3D modeling of molecules/reactions using Three.js and Mol* (e.g., rotate insulin PDB 1TRZ).
   - Physics/chemistry accuracy via Rapier.js (collisions) and numeric.js (kinetics solvers).
   - Biology pathways (e.g., glycolysis flux) with step-by-step animations.

2. **AI Integration**:
   - Built-in "Science ASimulation" AI (prompt-based, like the ones we discussed) for guidance: "What if I add HCl to NaOH?" ‚Üí Instant stoichiometry, pH calc, visual sim.
   - Error detection: Gentle feedback (e.g., "Enzyme denatured at 85¬∞C‚Äîtry cooling!").

3. **Data Visualization & Export**:
   - Auto-generated charts (Plotly.js) for kinetics curves, titration plots, growth data.
   - Lab reports: Downloadable PDFs with observations, equations (KaTeX-rendered), and raw data.

4. **Gamification**:
   - Badges, points, leaderboards; quests like "Complete 5 reactions for Biochemistry Badge."
   - Multiplayer mode (beta): Collaborate on sims in real-time.

5. **Accessibility Tools**:
   - Voice narration; high-contrast mode; keyboard navigation.
   - Offline mode (caches basic sims via service workers).

6. **Admin/ Educator Tools**:
   - Class creation; assignment tracking; bulk analytics (e.g., class avg. on PCR sim).

### Access and User Levels

Access is tiered by profile, ensuring content matches skill level. Upon signup, users take a 5-min assessment quiz to assign a baseline level (updated via activity).

| User Level | Access Granted | Examples |
|------------|----------------|----------|
| **Beginner (Novice)** | Basic sims; guided tutorials; no advanced reactions. | Simple acid-base mixing; cell structure views; 10-min sessions. |
| **Intermediate (Learner)** | Full catalog; quizzes; workspaces. | Enzyme assays; PCR runs; custom mixtures (e.g., 0.1M NaOH + HCl). |
| **Advanced (Expert)** | All features; custom scripting (JS snippets for reactions); API access. | Complex pathways (Krebs cycle modeling); export models to Unity; peer review tools. |
| **Educator/Admin** | + Class management; analytics dashboard; custom module creation. | Assign levels per student; generate reports; integrate with LMS. |

- **Login**: Free via email/Google; guest mode for public sims (limited saves).
- **Restrictions**: Premium for unlimited runs; schools get SSO (Single Sign-On).
- **Progression**: Earn XP to level up; e.g., complete titration ‚Üí unlock redox sims.

### Profile System

Profiles are central to personalization, stored in Supabase DB.

- **Setup**: Email + quiz ‚Üí auto-generates avatar, level, and starting dashboard.
- **Features**:
  - **Bio Tab**: Skills checklist (e.g., "Proficient in SMILES notation"); photo upload.
  - **Progress Tab**: Heatmap of completed sims; skill trees (e.g., branch to "Organic Chem").
  - **Settings**: Notifications (e.g., "New sim alert"); data export; level re-quiz.
  - **Social**: Follow users; share profiles for collaborations.
- **Privacy**: Opt-in sharing; anonymized analytics.

### Dashboard

The `/dashboard` is a dynamic hub, loading based on profile level. It uses React for real-time updates.

- **Layout**: Top: Welcome + level badge. Middle: Tiles for Recent Activity, Recommendations, Quick Start. Bottom: Stats graphs.
- **Personalization**:
  - Beginner: Simple cards (e.g., "Try Your First Reaction").
  - Advanced: Code snippets, API keys.
- **Widgets**:
  - Activity Feed: "Completed: Michaelis-Menten sim (Score: 92%)".
  - Recommendations: AI-suggested sims (e.g., "Next: Glycolysis after DNA basics").
  - Analytics: Weekly hours logged; completion rates.
- **Integrations**: Embed Google Calendar for scheduled labs; export to Notion.

### Workspaces

Workspaces (/workspaces) act as "personal labs" for ongoing projects, like Google Drive for science.

- **Creation**: New workspace ‚Üí Name it (e.g., "Enzyme Project") ‚Üí Add sims/files.
- **Functionalities**:
  - **Folders**: Organize by topic (e.g., /chemistry/reactions).
  - **Collaboration**: Invite users (role: Viewer/Editor); real-time co-editing (via Supabase Realtime).
  - **Storage**: Unlimited for free (100MB premium); supports PDB files, SMILES strings, data CSVs.
  - **Versioning**: Git-like history; fork sims.
  - **Level-Based**: Beginners get templates; Experts can script custom utilities (e.g., auto-pH calc).
- **Examples**: Save a titration setup ‚Üí Replay with variations ‚Üí Share report.
